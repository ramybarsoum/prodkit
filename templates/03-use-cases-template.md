> **What this doc is:** A PM working document for mapping out the full workflow before Feature Docs are written. Use it to think through scenarios, steps, and acceptance criteria while scope is still forming.
>
> **What happens to this doc:** Once you write the Feature Doc, copy all use cases into Section 5 of the Feature Doc. That's where they live permanently. Use cases in two places go out of sync — so this doc gets superseded. The Feature Doc is the source of truth.
>
> **We say "use cases" — not "user stories."** Each use case is a specific scenario (who is doing what and why), a step-by-step interaction flow, and binary acceptance criteria: PASS or FAIL, no ambiguity.

## Quality Check

- [ ] Every major workflow path has a use case
- [ ] Every use case has a specific scenario — who, what, why (not "a user wants to...")
- [ ] Steps are concrete actions, not vague states
- [ ] Acceptance criteria are binary PASS/FAIL — not "works correctly" or "displays properly"
- [ ] Actors are specific (not "users" — which users, doing what?)
- [ ] Out-of-scope use cases are documented with reasons
- [ ] All use cases are ready to copy into Section 5 of the Feature Doc
